# 碎片存档系统 v2 — 最终方案

**状态：✅ 方案确认，待实现**

---

## 一、核心设计目标

把"对话中无意中产生的有价值想法"自动存档，不需要用户主动触发。同时避免丢失主要任务的背景信息。

---

## 二、自动判断机制

### 触发信号 + 置信度分级

| 置信度 | 信号 | 示例 | 是否存档 |
|--------|------|------|---------|
| 高 | 问了一个很基础但很关键的问题 | "什么叫 context window？" | ✅ 直接存 |
| 高 | 说了自己的理解或总结 | "所以肠-皮轴就是肠道不好皮肤也会发炎？" | ✅ 直接存 |
| 高 | 对某个方案表现出特别兴趣 | 多追问了 2 次以上 | ✅ 直接存 |
| 低 | 只是多聊了几句 | 正常对话延伸但没有深入 | ⚠️ 不超过日上限时存 |
| 低 | 顺嘴提了一个新方向 | "对了，这个能不能用在 xxx 上？" | ⚠️ 不超过日上限时存 |

### 日上限
- 每天最多存 **10 条**碎片
- 超出时按置信度排序，高置信度优先
- **绕过机制**：用户主动说"这条很重要，存一下"→ 不受日上限限制

### 来源过滤
| session key | 类型 | 存碎片？ |
|-------------|------|---------|
| `agent:main:main` | webchat | ✅ |
| `agent:main:discord:channel:xxxxx` | Discord 频道（guild `1407234849445642270`） | ✅（guild 仅 2 人） |

## 三、碎片存档结构

```json
{
  "lastUpdated": "2026-04-05",
  "fragments": [
    {
      "id": "uuid",
      "timestamp": "2026-04-05 14:38",
      "content": "你对某个话题表现出意外的兴趣...",
      "topics": ["碎片机制", "自动存档"],
      "confidence": "high",
      "source": "auto",
      "mainContext": "毛囊炎补剂方案 → 突然问：肠-皮轴"
    }
  ]
}
```

**mainContext 格式：** 结构化箭头格式 `话题A → 话题B（碎片触发点）`
- 不依赖 AI 自由文本生成，质量更稳定

---

## 四、碎片缓冲区 + heartbeat 批量存档

- 对话过程中，碎片进入**内存缓冲区**（不实时写文件）
- 每次 heartbeat 时，批量写入碎片存档
- **碎片通知**：不立即打断，改为**下次用户发消息时顺便展示**
  - "今天新增了 3 条碎片：[摘要...]"
  - 如果用户最近一条消息距离现在 < 5 分钟（还在聊），跳过本次通知

---

## 五、Session 启动时加载

每次 `/new` 或 `/reset`：
```
chat-memory.ps1 -Action loadrecent -Limit 10
```
- 默认返回最新 **10 条**碎片摘要（topics + 首句 + 时间）
- 显示总数："近 3 天共 X 条碎片，可搜索查看详情"

---

## 六、碎片搜索

`searchfragments -Query "肠-皮轴 补剂"`：
- AND 逻辑（同时满足所有关键词才显示）
- 匹配范围：content + topics
- 结果按时间倒序，展示 mainContext

格式：
```
🔍 找到 2 条相关碎片：

[04-04] #肠-皮轴 #毛囊炎
你：「所以肠-皮轴就是肠道不好皮肤也会发炎？」
背景：毛囊炎补剂方案 → 突然问：肠-皮轴
---
[04-03] #补剂 #过敏
你：「VD3 4000IU 会心跳加快是什么原理？」
背景：讨论 Jamieson D3 vs Now D3 剂量
```

## 七、反悔机制

| 命令 | 说明 |
|------|------|
| `removefragment -Query "关键词"` | 先把 buffer 内容 flush 到硬盘，再删除匹配到的碎片 |
| `keepfragment -Query "关键词"` | 晋升为永久（不参与 7 天清理） |

---

## 八、晋升机制

碎片被搜索/查看后，自动延长到 **30 天**（从查看当天起算）。
