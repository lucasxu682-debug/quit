# 长时间任务进度反馈机制

## 目的
在执行超过5分钟的任务时，定期向用户报告进度，避免用户不知道任务状态。

## 机制

### 1. 任务开始时
- 记录开始时间
- 估算任务总时长
- 设定检查点（每5分钟）

### 2. 执行过程中
每5分钟自我检查：
- 已完成的工作量
- 剩余工作量
- 预计完成时间
- 向用户报告进度

### 3. 报告格式
```
【进度更新 - 已运行X分钟】
✅ 已完成: [具体完成的内容]
⏳ 进行中: [当前正在做的]
📊 进度: X%
⏱️ 预计剩余: X分钟
```

## 实施方式

由于我无法自主定时唤醒，采用以下策略：

### 策略：任务分块 + 主动报告
将长任务分解为多个阶段，每个阶段完成后检查时间：

```
开始任务
↓
记录开始时间
↓
执行阶段1
↓
检查: 是否已过5分钟? 
  → 是: 报告进度
  → 否: 继续
↓
执行阶段2
↓
检查: 是否已过5分钟?
  → 是: 报告进度
  → 否: 继续
↓
...重复直到完成
```

## 应用示例

### 示例1: 文件清理脚本编写
```
[开始] 预计需要10-15分钟

[5分钟后]
【进度更新 - 已运行5分钟】
✅ 已完成: 批处理脚本基础框架
⏳ 进行中: 编写清理函数
📊 进度: 40%
⏱️ 预计剩余: 7分钟

[10分钟后]
【进度更新 - 已运行10分钟】
✅ 已完成: 批处理脚本 + PowerShell脚本初版
⏳ 进行中: 添加保护机制
📊 进度: 75%
⏱️ 预计剩余: 3分钟
```

### 示例2: 大文件分析
```
[开始] 预计需要20分钟

[5分钟后]
【进度更新 - 已运行5分钟】
✅ 已完成: 读取前1000行
⏳ 进行中: 分析数据结构
📊 进度: 25%
⏱️ 预计剩余: 15分钟
```

## 规则

1. **只在长时间任务中使用** (>5分钟)
2. **每5分钟报告一次**，不提前
3. **报告内容要具体**，不要泛泛而谈
4. **给出预计完成时间**
5. **如果卡住要说明原因**
